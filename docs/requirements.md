# Требования к мини-проекту

## Название
Book Tracker

## Проблема
Пользователь хранит список книг в голове, в заметках или в разрозненных файлах и быстро теряет статус чтения, оценки и заметки. Приложение собирает эти данные в одном месте.

## Целевая аудитория
- Студенты
- Читатели, которые ведут личный список книг
- Пользователи, которым нужен простой трекер без сложной регистрации

## Функциональные требования
1. Просмотр списка книг.
2. Добавление новой книги через форму.
3. Просмотр карточки книги.
4. Редактирование книги.
5. Удаление книги.
6. Поиск по названию, автору или жанру.
7. Фильтрация по статусу чтения.

## Нефункциональные требования
- Приложение работает на Flask.
- Данные сохраняются в SQLite.
- Формы используют POST-запросы.
- Код разделён на модули.
- Есть автоматические тесты на основные сценарии.

## Архитектура
- Браузер
- Flask-приложение
- SQLite-база данных
