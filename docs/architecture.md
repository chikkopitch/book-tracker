# Архитектура

## Слои
1. `routes.py` принимает HTTP-запросы и возвращает HTML-страницы.
2. `services.py` содержит бизнес-логику, валидацию и операции с книгами.
3. `db.py` отвечает за подключение к SQLite и инициализацию схемы.
4. `templates/` формирует HTML.

## Маршруты
- `/` — список книг, поиск и фильтр.
- `/books/new` — добавление книги.
- `/books/<id>` — детальная страница.
- `/books/<id>/edit` — редактирование.
- `/books/<id>/delete` — удаление.

## Почему SQLite
SQLite подходит для небольшого учебного проекта: не требует отдельного сервера, легко запускается локально и хорошо подходит для хранения нескольких таблиц.

## Почему такая структура
Разделение на маршруты, сервисы и слой БД упрощает чтение кода и делает тестирование удобнее.
